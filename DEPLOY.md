# B2B Stickers — Deployment Guide

## What I Already Did For You

All the code-level work is complete:

- **Admin authentication system** — bcrypt password hashing, JWT tokens, HTTP-only cookies
- **Protected API routes** — all admin endpoints require authentication
- **Dashboard login page** — clean UI with password visibility toggle
- **Auth guard** — unauthenticated users redirected to `/login`
- **Logout functionality** — clears session, redirects to login
- **CORS security** — localhost only allowed in development mode
- **Dockerfile** — uses `$PORT` env var for Railway compatibility
- **Railway config** — `railway.json` with health checks
- **Procfile** — web + worker + beat processes
- **GitHub Actions CI** — tests API imports, dashboard build, site build
- **Sentry integration** — error tracking (just add `SENTRY_DSN`)
- **Pre-deploy health check** — `scripts/deploy_check.py`
- **Site build fix** — Suspense boundary for `useSearchParams`

## What You Need To Do (Step by Step)

### Step 1: Push Code to GitHub

```bash
# Make sure you're in the project root
cd /path/to/b2b-stickers

# Initialize git if not already
git init
git add .
git commit -m "Ready for production deployment"

# Create a new GitHub repo, then:
git remote add origin https://github.com/YOUR_USERNAME/b2b-stickers.git
git branch -M main
git push -u origin main
```

### Step 2: Sign Up for Services

| Service | URL | What You Need |
|---------|-----|---------------|
| **Railway** | https://railway.app | Account (free tier) |
| **Stripe** | https://dashboard.stripe.com/register | Account + API keys |
| **Resend** | https://resend.com | Account + API key |

### Step 3: Generate Your Admin Password

```bash
cd apps/api
python scripts/hash_password.py "your-secure-admin-password"
```

**Copy the output hash.** You'll paste it into Railway in Step 5.

### Step 4: Create Railway Project

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login
railway login

# Create project from your GitHub repo
cd apps/api
railway init
# Select "Deploy from GitHub repo"
# Choose your b2b-stickers repo
# Set root directory to: apps/api
```

### Step 5: Add PostgreSQL + Redis

In Railway Dashboard:
1. Click "New" → Database → Add PostgreSQL
2. Click "New" → Database → Add Redis

### Step 6: Set Environment Variables

In Railway Dashboard → Your API Service → Variables:

**Click "Raw Editor" and paste this (fill in your values):**

```env
# Auto-generated by Railway (don't change)
DATABASE_URL=${{Postgres.DATABASE_URL}}
REDIS_URL=${{Redis.REDIS_URL}}
CELERY_BROKER_URL=${{Redis.REDIS_URL}}
CELERY_RESULT_BACKEND=${{Redis.REDIS_URL}}

# Auth (REQUIRED)
JWT_SECRET=<paste: openssl rand -hex 32>
ADMIN_PASSWORD_HASH=<paste from Step 3>

# Stripe (REQUIRED)
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_SUBSCRIPTION_PRODUCT_ID=prod_...

# App URLs (update after deploying frontend)
API_BASE_URL=https://your-api.up.railway.app
SITE_BASE_URL=https://your-site.vercel.app
DASHBOARD_BASE_URL=https://your-dashboard.vercel.app
ENV=production

# Email (CORE)
RESEND_API_KEY=re_...
EMAIL_FROM=B2B Stickers <hello@b2b-stickers.co.uk>

# VAT (CORE)
VAT_RATE_PERCENT=20.0
VAT_NUMBER=GB123456789
```

### Step 7: Deploy the API

```bash
cd apps/api
railway up
```

Grab your API URL:
```bash
railway domain
# Copy: https://b2b-stickers-api.up.railway.app
```

### Step 8: Initialize the Database

```bash
# Connect to Railway Postgres
railway connect postgres

# Run inside psql:
\i ../../../scripts/init_db.sql
\i ../../../scripts/seed_industries.sql
\i ../../../scripts/seed_pricing_matrix.sql
\q
```

### Step 9: Create Alembic Migration

```bash
cd apps/api
railway run -- alembic revision --autogenerate -m "initial"
railway run -- alembic upgrade head
```

### Step 10: Seed Demo Products

```bash
cd apps/api
railway run -- python scripts/seed_demo_products.py --count 50
```

### Step 11: Configure Stripe Webhooks

In Stripe Dashboard → Developers → Webhooks:

1. Click "Add endpoint"
2. **Endpoint URL:** `https://your-api.up.railway.app/api/v1/webhooks/stripe`
3. Select events:
   - `checkout.session.completed`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
   - `invoice.paid`
   - `invoice.payment_failed`
4. Click "Add endpoint"
5. Copy the **Signing secret** (starts with `whsec_`)
6. Paste it into Railway as `STRIPE_WEBHOOK_SECRET`
7. Redeploy: `railway up`

### Step 12: Deploy Dashboard to Vercel

```bash
cd apps/dashboard

# Install Vercel CLI
npm install -g vercel

# Deploy
vercel --prod
```

In Vercel Dashboard → Project Settings → Environment Variables:
```
API_BASE_URL=https://your-api.up.railway.app
```

### Step 13: Deploy Site to Vercel

```bash
cd apps/site
vercel --prod
```

In Vercel Dashboard → Environment Variables:
```
API_BASE_URL=https://your-api.up.railway.app
NEXT_PUBLIC_SITE_URL=https://your-site.vercel.app
```

### Step 14: Update URLs & Redeploy API

Go back to Railway Dashboard and update:
```
SITE_BASE_URL=https://your-site.vercel.app
DASHBOARD_BASE_URL=https://your-dashboard.vercel.app
```

Redeploy:
```bash
cd apps/api && railway up
```

### Step 15: Verify Everything

```bash
# Health check
curl https://your-api.up.railway.app/health

# Test login
curl -X POST https://your-api.up.railway.app/api/v1/admin/login \
  -H "Content-Type: application/json" \
  -d '{"password":"your-admin-password"}' \
  -c cookies.txt

# Test protected endpoint
curl https://your-api.up.railway.app/api/v1/admin/me \
  -b cookies.txt
```

Open the dashboard in your browser:
```
https://your-dashboard.vercel.app/login
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `401` on dashboard | Check `ADMIN_PASSWORD_HASH` is set correctly in Railway |
| `CORS error` | Verify `DASHBOARD_BASE_URL` matches your actual Vercel URL |
| `Database connection failed` | Check `DATABASE_URL` — Railway provides this automatically |
| Stripe webhooks failing | Make sure webhook URL uses HTTPS, not HTTP |
| `Module not found` on deploy | Ensure all `__init__.py` files are committed to git |
